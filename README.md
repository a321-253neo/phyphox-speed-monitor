本工具基于phyphox官方的“Camera stopwatch: Brightness”实验改造而来。
测量亮度发生变化的时间间隔，例如由物体经过引起的变化。并借助用户输入的物体间距近似地计算出这一过程中的瞬时速度。请注意：实际运动速度越快，测量误差越小。
此配置允许你为测量矩形区域内的平均亮度（更具体地说是光亮度）设置一个下限阈值和一个上限阈值。如果亮度超过上限阈值或低于下限阈值，当前时间将被记录为一个触发事件。系统会记录不同事件之间的时间间隔以及这些事件的持续时间。
时间分辨率由摄像头的预览帧率决定，通常固定为 30fps，因此采样率为 33 毫秒。请注意，Phyphox 使用的是摄像头的预览流进行实时分析，其速率受限于此。如果你的手机支持高速或慢动作录制，更高的帧率仅适用于录制模式（而非实时模式），而 Phyphox 目前并未使用录制模式。
如果你使用自动曝光，请注意平均亮度会在短时间内漂移到 0.5 左右。这可能允许在光线变化的条件下进行可靠的触发，但也可能会影响触发和特别是测量持续时间的精度。另外请注意，此秒表使用的是亮度（luma）而非光亮度（luminance），即一种可以更快计算出的亮度的非线性表示。这意味着该值将无法与光亮度测量值进行比较，并且会受到曝光设置变化的影响。
如果你愿意减小自动曝光带来的误差，你可以点击摄像头预览框并将下方的“AE”设置为“off”
